Session Variable 시작  다음  이전

SLEE가 제공하는 각종 설정정보들을 가지며, 여기에 서비스가 처음 실행될 때 발생된 메시지 정보, 채널정보 등이 있습니다.

 

사용방법은 "session." 을 입력하면 7 개의 타입(call, command, cti, daemon, exception, menu, sys) 이 제공되며, 다시 각 타입별로 변수가 제공됩니다.(예 : session.call.ani)

 

 

[ sys ]

 

Name Description
session.sys.trunkid MCE(TE) system ID
session.sys.channel LI( Driver Logical Index ) 값으로, SLEE 채널 번호로 사용된다
session.sys.physicalline MCE physical channel
session.sys.ipaddr System ip address
session.sys.orgsysid 최초 호가 인입된 System ID
session.sys.sysid 현재 호가 진행중인 System ID
session.sys.roleid 현재 호가 진행중인 시스템의 Role ID
session.sys.nodeid Node ID

[ sip ]

 

Name Description
session.sip.header

인바운드시 들어온 SIP Relay Header

사용법은 header를 배열 변수로 사용, 배열 인덱스는 SIP Header Name

ex) session.sip.header['Call-ID'], session.sip.header['CSeq']

* ForCus v5.2.0-b3 이상 지원

 

 

[ lib ]

 

Name Description
session.lib.result

라이브러리 서버 Java script, Python, Shell Script 호출 처리 결과

- JavaScript : 호출 된 Function 내부에서 return 되는 결과값
- Python : 호출 된 Python 라이브러리에서 print 로 출력 되는 모든 내용
- Shell Script : 호출 된 Shell Script 에서 echo 로 출력 되는 모든 내용

* ForCus v5.3 이상 지원
