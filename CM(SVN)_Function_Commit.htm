Commit

 

- 시나리오 수정, 추가, 삭제, 이름 변경 등 시나리오 파일이 변경 되었을 때 Commit 을 해야 원격 저장소(Repository) 에도 적용 됩니다.

 

 

Local directory : 형상관리 중인 Working copy 디렉토리(대부분은 솔루션 디렉토리)

Remote repository : 원격 저장소 URL 주소

 

Commit message : Commit 시에 시나리오 수정 내용 등을 입력하여 추 후 해당 Revision(Commit 시에 부여되는 관리 번호) 이 어떤 내용이 수정되었는지 알 수 있습니다.

 

Changes List : Commit 대상 파일 리스트 이며 왼쪽의 체크 박스를 체크하여 대상에서 제외할 수 있습니다.

Show unversioned files Checkbox : NotVersioned 파일을 모두 Changes List에 리스팅 하고 체크 박스로 체크 한 후 Commit 하면 해당 파일을 바로 Add 및 커밋 합니다.

 

Keep locks Checkbox : 파일에 SVN Lock 을 건 상태에서 Commit 시에 Lock이 풀리는 것을 방지 합니다.

 

Show log Button : Remote repository URL 주소에 해당하는 원격 저장소에 Commit된 Revision 리스트를 볼수 있습니다.

ex) 위 그림에서 처럼 URL 주소 /trunk 에 Commit된 Revision 리스트를 볼수 있습니다.

 

* 디렉토리를 선택하면 하위의 Commit 대상 파일들이 Changes List 에 리스팅 되고, 파일을 선택하면 선택한 파일 만 리스팅 합니다.

 

* SCE 리본 메뉴의 Commit은 솔루션의 모든 Commit 대상 파일들을 리스팅 합니다.
