ServiceDataSync 시작  다음  이전

ForCus SLEE 이중화 시에 시나리오 서비스 데이터(변수) 동기화를 수행하는 아이템 입니다.

 

SLEE 이중화란 서비스 장비 n대에 SLEE 모듈(SLEE0, SLEE1)이 있는데, 장비에 문제가 있거나, 또는 모듈, 네트워크등 해당 SLEE에 문제가 발생할 경우 Standby중인 백업 SLEE서버로 절체를 하고 호가 끊어지지 않고 서비스 중이던 메뉴까지 복원하는 기능을 말합니다.

 

SLEE 서비스 데이터는 UCID를 기준으로 백업 되며, 현재는 n대의 SLEE 서버와 1대의 백업 서버의 구조로 연동됩니다.

 

ServiceDataSync는 데이터 동기화를 수행하는 아이템 이지만 복구 콜인 경우에는 서비스 복원지점인 Check Point역할을 합니다. 마지막으로 정상적으로 동기화 된 부분부터 복원이 됩니다.(현재는 메뉴의 각 시작지점이 Data Sync부분 겸 Check Point가 됩니다.)

 

제약사항 : 현재는 전역변수에 대해서만 데이터 동기화(백업서버)가 가능합니다.

 

* 이 아이템은 아직은 사용할 수 없으며, DefineMenu에서는 각 메뉴별로 자동으로 생성됩니다.(추후 로컬변수 동기화가 지원될 때 사용합니다.)
