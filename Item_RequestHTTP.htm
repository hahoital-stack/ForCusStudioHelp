RequestHTTP 시작  다음  이전

HTTP 프로토콜을 이용하여 전문을 보내 REST API 연동을하는 아이템이다.

 

* ForCus v5.3.3-b1 이상 지원

 

ICON

 

 

 

PROPERTIES

 

RequestHTTP Info

 

 

Name : 아이템의 이름을 지정 합니다. 지정된 이름은 Packet 처리를 위한 변수에 접근 할 때 사용 합니다.

 

Timeout : 전문 전송 시 대기 시간을 지정 합니다.(second 단위)

 

Choice Call(for BGM) : Multi Call 에서 콜을 선택 합니다.(2개 콜 까지 가능, GetChannel 로 생성)

 

BGM : 전문 전송 시 Delay 가 발생 할 수 있으므로 배경 음악을 지정 하여 전문 전송 중 재생 합니다.

 

Condition : 아이템의 수행 조건을 지정 합니다.

 

 

Packet Data

 

 

Json형식 전송 전문을 입력 합니다.

 

PackJsonEx 아이템과 같이 " . " 으로 구분하여 다차원의 형식으로 작성할 수 있습니다.

 

Name부분에 -(minus) 기호를 사용하려면 변수 규칙때문에 사용이 불가하나 다음방법으로 사용가능 합니다.

Ex) Header 의 User-Agent 를 작성하는 경우
['User-Agent'] = 'Bridgetec Forcus' 과 같이 작성합니다.

 

배열 형식은 example[1], example[2] 형식으로 작성합니다.

 

Request URL : 요청할 URL의 전체 주소 또는 변수를 입력합니다.

 

Method : GET, POST, PUT, PATCH, DELETE, OPTIONS 중 선택합니다.

 

Content Type : HTTP의 Content Type을 지정합니다. (application/json 고정, 추후 확장)

 

Connection Timeout : HTTP 서버연결 Timeout (단위 : 초)

 

HTTP Timeout : 연결 후 HTTP 응답 Timeout (단위 : 초)

 

Verify Certification : HTTPS 프로토콜 사용시 인증서 무시 여부 (Check : 인증서 일치해야 작동, UnCheck : 인증서 일치하지 않아도 작동)

 

Head Data : HTTP의 Head 데이터를 지정합니다.

 

Body Data : HTTP의 Body 데이터를 지정합니다.

 

Using Expression : 변수에 대입 되어 있는 json데이터를 사용 할때 해당 변수의 이름을 입력(체크시 Body 데이터 비활성화)

 

Up Button : 선택한 전문 필드를 위로 이동 합니다.

 

Dn Button : 선택한 전문 필드를 아래로 이동 합니다.

 

View Button : 리스트의 내용을 팝업창으로 Json 형식화하여 보여줍니다.

 

X Button : 선택한 리스트를 잘라냅니다.(Ctrl+X)

C Button : 선택한 리스트의 내용을 복사 합니다.(Ctrl+C)

V Button : 복사한 리스트의 내용을 붙여넣기 합니다.(Ctrl+V)

 

E Button : 선택한 전문 필드의 내용을 수정 합니다.

 

+ Button : 전문 필드 하나를 추가 합니다.(PacketEx Info Tab 의 Packet ID 에 지정한 정의된 전문에 포함된 필드)

 

- Button : 선택한 전문 필드를 삭제 합니다.

 

 

Tracking

 

 

전문 처리에 대한 트래킹을 처리합니다.

 

Properties

 

Check Tracking Check Box : Tracking을 사용 할 것인지를 체크합니다.

Tracking Point Flag Check Box : 특별한 Tracking Flag를 설정하여 SWAT에서 트래킹 시에 이 플래그가 설정된 항목만 트래킹 할 수 있습니다.

Encryption Value1 Check Box : Track Value1 속성 데이터에 대해서 암호화 처리를 합니다.

 

Track ID : 정의한 Track ID를 선택합니다.

- DefineTrack Item에 정의된 Track ID가 콤보박스 리스트로 제공됩니다.(Type이 없거나, Packet인 경우)

* 기본으로 Packet ID가 설정됩니다.

 

Track Name : Track ID를 선택하면 같이 정의된 이름을 출력합니다.(SWAT에서 이 이름으로 트랙킹 시에 사용합니다.)

 

Track Value1 : 트래킹 시에 사용할 값을 입력 합니다.

 

Track Value2 :트래킹 시에 사용할 값을 입력 합니다.(결과에 대한 Reason값)

 

 

RESULT

 

· 처리 결과 세션 변수

 

session.command.result : 처리 성공(_SLEE_SUCCESS_), 실패(_SLEE_FAILED_)

session.command.reason : 실패면 실패 오류코드(시스템 상수 참조)

 

· 아이템 속성 변수(@name은 Name항목에서 지정한 이름)

 

Json Packet 발신 및 수신 전문의 값 참조는 SLEE 환경변수의 PACKETJSON_RESULT_SET_FROM 설정(기본 설정 YES)에 따라 아래와 같습니다.

 

- PACKETJSON_RESULT_SET_FROM=NO, Asis 시나리오 호환 시 설정

 

@name.header.항목명 : 발신, 수신 모두의 Header 값을 참조 하기 위한 변수, RequestHTTP 아이템 입력(발신) 항목 및 수신 시 수신항목 생성

 

@name.body.항목명 : 발신, 수신 모두의 Body 값을 참조 하기 위한 변수, RequestHTTP 아이템 입력(발신) 항목 및 수신 시 수신항목 생성

 

@name.to.header.항목명 : 발신 Header 값을 참조 하기 위한 전용변수, 발신 시 생성

 

@name.to.body.항목명 : 발신 Body 값을 참조 하기 위한 전용변수, 발신 시 생성

 

※ 주의

- @name.header(body).항목명 사용 시 발신항목과 수신항목이 중복되는 경우 수신된 값으로 갱신됩니다.

 

- 전문 재 발신 시 @name.header(body) 변수 초기화 후 발신 전문항목의 값을 재 할당하지 않으면 발신값이 수신값으로 갱신되거나 수신된 전문항목이 발신항목에 추가되어 발신될 수 있습니다.

이 경우 간단히 발신전용 변수 값을 @name.header(body)변수에 재 할당 한 후 발신하면 됩니다.( ex) @name.header = @name.to.header, @name.body = @name.to.body )

 

- PACKETJSON_RESULT_SET_FROM=YES

 

@name.header.항목명 : 발신 Header 값을 참조 하기 위한 변수, RequestHTTP 아이템 입력(발신) 항목

 

@name.body.항목명 : 발신 Body 값을 참조 하기 위한 변수, RequestHTTP 아이템 입력(발신) 항목

 

@name.to.header.항목명 : 발신 Header 값을 참조 하기 위한 전용변수, 발신 시 생성

 

@name.to.body.항목명 : 발신 Body 값을 참조 하기 위한 변수, 발신 시 생성

 

@name.from.header.항목명 : 수신 Header 값을 참조 하기 위한 변수, 전문 수신 시 생성

 

@name.from.body.항목명 : 수신 Body 값을 참조 하기 위한 변수, 전문 수신 시 생성

 

 

※ SLEE Cfg 경로 : /bridgetec/ipron/ir/cfg/irslee.cfg

 

 

 

수신 Header수신 Body

TestHTTP.header.Referer

 

TestHTTP.body.company

TestHTTP.from.header.Referer

 

TestHTTP.from.body.company
