ICON

 

 

 

PROPERTIES

 

Call Info

 

 

Name : 아이템의 이름을 지정 합니다.

 

To : 전화를 걸 전화번호를 지정 합니다.

 

ANI : 전화를 걸때 설정 할 발신자 전화번호를 지정 합니다.

 

User Data : 사용자 데이터를 입력 합니다.

 

User Data Type : 사용자 데이터 타입을 선택 합니다.(Hex String, Normal String)

 

Timeout : 전화 연결 대기시간을 설정 합니다.

 

Tone Detect Check Box : 발신 시도 후 btmp 에 등록된 멘트 감지 사용 유무 (Busy, 없는 전화번호, 무응답 등)

 

Tone Dial Timeout : 전화 연결시 타임아웃(Busy, Ringing등을 인지하는 타임아웃)

 

Tone Timeout : 전화 연결 후 타임아웃(사람이 받았는지 기계(팩스등)가 받았는지 인지하는 타임아웃)

 

Agent Call Check Box : 상담원 연결 카운트 집계를 위해서 체크한다.(하나의 콜에는 카운트가 1번만 집계된다. => v5.1 이후 : CtiCall 아이템 1회 호출 당 콜 통계 집계 1회 추가)

 

Agent Type : 연결 상담원 타입을 지정합니다. 상담원 연결 통계에 사용되며, Agent Call이 체크될 때만 활설화 됩니다.

Agent(상담원), Branch office agent(영업점 상담원), Reserve1(예비1), Reserve2(예비2), Reserve3(예비3)

 

Media Type : 미디어 타입을 설정합니다.

 

Connected Mask : Connected Mask속성을 설정합니다.(전화를 걸때 연결 성공으로 처리하는 마스크 값)

PROCEEDING: 전화를 걸기를 시작하면 성공

RING_BEGIN: 링이 시작되면 성공

CON_SIGNAL: 전화기가 주는 신호를 받으면 성공(Off-Hook)

VOICE_BEGIN: 목소리 감지가 되면 성공

ANSWER(SIGNAL+VOICE): 전화기가 주는 신호와 목소리가 감지되면 성공

Default: 시스템 기본설정

 

DN Group Name : 전화를 걸 DN 그룹을 지정 합니다.(아웃 바운드 채널 그룹)

Mute Transfer 기능 사용 시 "LOCALLINE" 으로 지정 합니다.

 

Choice Call : Multi Call 에서 콜을 선택 합니다.(2개 콜 까지 가능, GetChannel 로 생성)

Mute Transfer 기능 사용 시 'Second' 를 선택 합니다.

 

※ Mute Transfer : MakeCall 이 성공(session.command.result == _SLEE_SUCCESS_) 한 후 Transfer 아이템을 이용 하여 호를 전환 합니다.

 

SIP Relay Header : 설정한 Relay Header 배열 변수를 지정합니다.(Ipron v5.2.0-b3, v5.3 이상 부터 사용 가능)

 

Assign 아이템으로 다음과 같이 Relay Header 여러 개를 설정 할 수 있습니다.

 

 

 

Tracking

 

 

트래킹을 처리합니다.

 

Properties

 

Check Tracking Check Box : Tracking을 사용 할 것인지를 체크합니다.

Tracking Point Flag Check Box : 특별한 Tracking Flag를 설정하여 SWAT에서 트래킹 시에 이 플래그가 설정된 항목만 트래킹 할 수 있습니다.

Encryption Value1 Check Box : Track Value1 속성 데이터에 대해서 암호화 처리를 합니다.

 

Track ID : 정의한 Track ID를 선택합니다.

- DefineTrack Item에 정의된 Track ID가 콤보박스 리스트로 제공됩니다.(Type이 없거나, MakeCall인 경우)

 

Track Name : Track ID를 선택하면 같이 정의된 이름을 출력합니다.(SWAT에서 이 이름으로 트랙킹 시에 사용합니다.)

 

Track Value1 : 트래킹 시에 사용할 값을 입력 합니다.

 

Track Value2 :트래킹 시에 사용할 값을 입력 합니다.(결과에 대한 Reason값)

 

 

RESULT

 

· 처리 결과 세션 변수

 

session.sys.trunkid : MCE(TE) system ID

session.sys.physicalline : MCE physical channel #

session.call.callid : MCE call ID로, MCE에서 콜마다 Unique하게 부여하는 ID(구성 : 프로세스 ID 4자리 + Channel ID 4자리+ 시분초(hhmmss) 6자리 + 랜던 6자리)

session.call.ani : MCE 또는 시나리오 전환 시 이전 시나리오로 부터 받은 ANI 값

session.call.dnis : MCE 또는 시나리오 전환 시 이전 시나리오로 부터 받은 DNIS 값

 

session.command.result : 처리 성공(_SLEE_SUCCESS_), 실패(_SLEE_FAILED_)

session.command.reason : 실패면 실패 오류코드(시스템 상수 참조)

session.call.agentresult : 상담원 연결 결과, 성공(_SLEE_SUCCESS_), 실패(_SLEE_FAILED_), 이 값은 수정할 수 있습니다.
