웹 페이지(HTML 파일)를 화면에 출력 하도록 스마트 폰으로 알림 메시지를 보냅니다.

 

 

ICON

 

 

 

PROPERTIES

 

Push Info

 

 

 

Name : 아이템 이름을 지정 합니다.(결과 변수에 접근할 때 사용 합니다.)

 

Transaction ID : TB 에서 전문을 전송할 대상(iWeb-Adapter)을 찾기 위한 서버 키 입니다.(ForCus v5.0.1 이상 부터는 필수 입력)

 

Push Type

 

Dummy Push Check Box : 내용이 없는 Push를 보내서 실제 스마트폰이 Push Receive가 가능인지 체크한다.

DTMF Push Check Box : 스마트폰에 다이얼 패드 창을 팝업 시킨다.

Popup Push Check Box : 스마트폰 화면 상단에 팝업창을 출력한다.

Native Push Check Box : SDE 작성한 JSON파일을 파싱하여 정보를 푸시한다.

Visual Push Check Box : IVR메뉴 멘트와 스마트폰 화면을 싱크하여 화면에 메뉴내용을 출력한다.

Timeout : Dummy Push, DTMF Push일 경우 Timeout을 설정합니다.

 

URL : 출력할 HTML URL 정보를 지정 합니다.

 

Message : 스마트폰 화면에 설정한 메시지가 출력됩니다.

 

TelNo : 출력할 스마트 폰의 전화번호를 지정 합니다.(알림 메시지를 위한 Device ID 를 찾기 위한 키)

 

DNI : 웹콜 서비스 시나리오를 위한 서비스 식별번호를 지정 합니다.(Dialed Number Identification)

 

Condition : 아이템의 실행 조건을 설정 합니다.

 

Hide push popup window Check Box : 푸시 요청시 팝업되는 윈도우를 이것을 체크하면 팝업하지 않습니다.(아이폰 전용)

 

Html Info : 현재 지원하지 않습니다.

 

 

Popup Push Info

 

 

Popup Title : 팝업 타이틀을 설정합니다.

 

Popup Message : 팝업 메시지를 설정합니다.

 

Gravity : 팝업 메시지 출력 위치를 선택합니다.

 

Use Vibration Check Box : 팝업 진동알림을 설정합니다.

 

Layout Type : Push 화면 배치를 선택합니다.

 

Background Color : Push 배경 색을 선택합니다.

 

Title Font Color : Push 타이틀 폰트 색을 선택합니다.

 

Title Font Size : Push 타이틀 폰트 크기를 설정합니다.(10 ~ 30 dpi)

 

Message Font Color : Push 메시지 폰트 색을 선택합니다.

 

Message Font Size : Push 메시지 폰트 크기를 설정합니다.(10 ~ 30 dpi)

 

Popup Time : 팝업 지속시간을 설정합니다.(Max 30000 ms)

 

Preview Button : 팝업 미리보기

 

 

Native Push Info

 

 

JSON Root Path : JSON 파일이 있는 로컬 디렉토리를 전체 경로로 표시 합니다

 

Load JSON Name : 선택한 JSON 파일 이름을 표시 합니다.

 

Load JSON Button : JSON 파일을 선택하면 내용을 파싱하여 ID, Screen ID, Version, Build 값을 읽어 옵니다.

 

JSON Meta Data

 

ID : JSON 파일이 빌드 될때 생성되는 Unique한 ID

 

Screen ID : Push Screen ID

 

Version : JSON 파일이 빌드 될때 생성되는 버전

 

Build : JSON 파일이 빌드 될때 생성되는 빌드 번호

 

 

Visual Push Info

 

 

Visual Push Title : Visual Push 타이틀을 설정합니다.

 

Visual Push Message Combo Box : Ment를 선택 하여 Description을 Visual Push Message Edit Box의 현재 커서 위치에 붙여 넣습니다.

 

Using Expression Check Box : Visual Push Message Edit Box의 내용을 연산식으로 처리 합니다.

 

• 선택 멘트 리스트

• Visual Push Message Edit Box

 

Preview Button : Visual Push 미리 보기

 

Del Button : Visual Push Message 콤보박스에서 멘트를 선택하면 멘트 리스트에 선택한 Ment와 Description이 추가되는데, 멘트 리스트에서 선택한 멘트를 제거한다.

 

Insert Message >> Button : 멘트 리스트에서 선택한 멘트의 Description을 Visual Push Message Edit Box의 현재 커서 위치에 붙여 넣습니다.

 

Visual Push Notify : Visual Push 알림을 설정합니다.

 

Layout Type : Push 화면 배치를 선택합니다.

 

Use Vibration Check Box : 팝업 진동알림을 설정합니다.

 

Background Color : Push 배경 색을 선택합니다.

 

Title Font Color : Push 타이틀 폰트 색을 선택합니다.

 

Title Font Size : Push 타이틀 폰트 크기를 설정합니다.(10 ~ 30 dpi)

 

Message Font Color : Push 메시지 폰트 색을 선택합니다.

 

Message Font Size : Push 메시지 폰트 크기를 설정합니다.(10 ~ 30 dpi)

 

Notify Font Color : Push 알림 폰트 색을 선택합니다.

 

Notify Font Size : Push 알림 폰트 크기를 설정합니다.(10 ~ 30 dpi)

 

 

Tracking

 

 

Push 요청한 데이터에 대한 트래킹을 처리합니다.

 

Properties

 

Check Tracking Check Box : Tracking을 사용 할 것인지를 체크합니다.

Tracking Point Flag Check Box : 특별한 Tracking Flag를 설정하여 SWAT에서 트래킹 시에 이 플래그가 설정된 항목만 트래킹 할 수 있습니다.

Encryption Value1 Check Box : Track Value1 속성 데이터에 대해서 암호화 처리를 합니다.

 

Track ID : 정의한 Track ID를 선택하거나 입력 합니다.

- DefineTrack Item에 정의된 Track ID가 콤보박스 리스트로 제공됩니다.(Type이 없거나, RequestPsuh인 경우)

 

Track Name : Track ID를 선택하면 같이 정의된 이름을 출력합니다.(SWAT에서 이 이름으로 트랙킹 시에 사용합니다.)

 

Track Value1 : 트래킹 시에 사용할 값을 입력 합니다.(값이 없으면 요청시 사용된 전화번호를 기본값으로 사용합니다.)

 

Track Value2 :트래킹 시에 사용할 값을 입력 합니다.(결과에 대한 Reason값)

 

 

RESULT

 

· 처리 결과 세션 변수

 

session.command.result : 처리 성공(_SLEE_SUCCESS_), 실패(_SLEE_FAILED_)

session.command.reason : 실패면 실패 오류코드(시스템 상수 참조)
