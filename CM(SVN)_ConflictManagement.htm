충돌 관리

 

형상관리 중인 디렉토리나 파일의 이름 변경, 삭제 시 에 다른 사용자는 CheckOut을 다시 받아야 할 수 도 있습니다.

 

윈도우 탐색기에서 형상관리 중인 디렉토리, 파일을 삭제, 이름 변경 등을 하면 Repository리는 변경이 안되기 때문에 형상관리 툴 등을 통해서 처리 해야 합니다.

 

* 형상관리 중인 디렉토리, 파일의 삭제, 이름 변경등은 신중히 해야 하며 같은 시나리오를 개발 중인 개발자들도 해당 항목을 확인하고 처리 해야 합니다.

 

 

SCE 시나리오 형상관리에서 가장 빈번하게 발생 할 수 있는 충돌 상황 및 원인

 

상황 원인
솔루션 파일 동시 변경

- 솔루션에 프로젝트를 추가, 삭제, 이름변경을 서로 다른 개발자가 동시에 하는 경우

 

=> 예방 : 추가, 삭제, 이름변경을 동시에 한 사람만 하고 Commit 까지 완료 후 다른 개발자는 Update 수행.(Update 하지 않고 위와 같은 작업은 충돌 발생)
프로젝트 파일 동시 변경

- 프로젝트에 시나리오 파일 추가, 삭제, 이름변경을 서로 다른 개발자가 동시에 하는 경우

 

- 프로젝트 Solution View, File View 트리 작업

   + 트리에 폴더 추가 / 삭제 / 이름변경

   + 시나리오 파일 설명 편집

 

- 리본 메뉴 Build > Build Settings 에서 Project Description 변경

 

=> 예방 : 같은 프로젝트를 서로 다른 개발자가 같이 개발 중이라면 위와 같은 상황은 동시에 한 사람만 하고 Commit 까지 완료 후 다른 개발자는 Update 수행.(Update 하지 않고 위와 같은 작업은 충돌 발생)

 

=> Commit 시에 프로젝트에 파일 추가, 삭제, 이름 변경 등은 프로젝트 파일이 변경되므로 반드시 프로젝트 파일(.sspx)도 Commit 해야 한다.
시나리오 파일 동시 변경

- 같은 시나리오 파일을 편집하고 Commit 하지 않고 있다가 다른 개발자가 편집 하고 Commit 한 후에 나중에 Commit 하는 경우

 

- 같은 시나리오 파일을 다른 개발자가 편집 하고 Commit 하였으나 Update 하지 않고 편집하고 Commit 하는 경우

 

=> 예방 : 같은 시나리오 파일은 서로 다른 개발자가 편집 금지, 편집 전에는 항상 Update 하기

[충돌 해결]

충돌이 발생한 시나리오 파일을 백업 후 삭제 한 다음 SVN Update 하면 저장소에 있는 최신 파일이 다운로드 된다.

SCE로 충돌이 발생한 시나리오를 열고 다시 추가 하고 자 하는 부분을 편집 한다.(이때 백업한 시나리오는 SCE를 하나 더 실행하여 열고 추가한 부분의 내용을 확인 할 수 있고 또는 아이템을 복사하여 붙여넣을 수 있다.)

 

• 기타 원인

- 형상관리 중인 디렉토리를 SVN Delete 하고 같은 이름으로 디렉토리를 만들어 Commit 한 상태에서 다른 사람이 Update 받을 때 tree conflict 발생

 

해결 : 해당 디렉토리를 삭제하고 SVN Update 합니다.

 

 

• 충돌 예제

- 같은 시나리오 파일을 다른 개발자가 편집 하고 Commit 하였으나 Update 하지 않고 편집하고 Commit 하는 경우

 

 

 

 

해결 : 위와 같은 에러 발생 시 윈도우 탐색기에서 수정한 파일들을 모두 백업 후 Conflicted 된 파일을 삭제하고 SVN Update를 실행하면 저장소에 있는 파일이 다운로드 되는데 이 파일을 편집 후 커밋합니다.
